# 朱雀 72 篇 A/B/C 收敛报告

## 一、计算口径

本报告把朱雀的 45 个连续标签区间投影到 72 篇匿名正文的字符区间中，五个空行分隔符不计入任何正文分母。正文总计 **70,953** 字符，其中人工特征 **14,878**、疑似 AI **56,075**、AI 特征 **0**，总体人工率为 **20.97%**。

PDF 文本抽取少了 50 个尾随空白字符，已按区间尾部归还前一篇。校验后总体人工率仍为 20.97%，与朱雀报告一致。单篇比例代表该文章在本次随机合并提交上下文中的标签分布，适合 A/B/C 同条件比较，但不等同于单独再次提交时必然得到完全相同的边界。

- B、C 的单篇人工率中位数都没有达到 60%。

## 四、最有价值的同场景证据

### CAMPUS-03：C 明确重复胜出

- A1/A2：24.78% / 22.90%
- B1/B2：29.62% / 58.63%
- C1/C2：70.52% / 80.56%

这是目前最强的 C 组证据。物件误认、追索、空口袋验证等场景，加入因果物件、被拒绝方案、代价和反制信息，确实把两次采样都推过了 60%。

### CAMPUS-04：A 明确重复胜出

- A1/A2：100% / 100%
- B1/B2：17.82% / 79.16%
- C1/C2：45.85% / 26.41%

完整 Planner 在“残缺消息 + 多人物 + 意图误读”场景中保留了 Brief 丢掉的信息。正确方向不是恢复全局 A，而是编译一个 **A-lite**：只补第二人物的目标、可见证据、可能反应和竞争性解释。

### CAMPUS-02：B 最稳定，C 上限最高但波动大

- A1/A2：52.32% / 34.53%
- B1/B2：60.06% / 77.93%
- C1/C2：100% / 22.14%

C 的四个字段能制造满分稿，但也可能把机制写得过于显眼。该类隐蔽报复场景优先 B，只有 Planner 的四项信息足够具体、非抽象时才升级 C。

### TASK-01：物理补救过程有信号

- A1/A2：0% / 61.44%
- B1/B2：49.64% / 54.06%
- C1/C2：72.20% / 28.07%

稳定性仍不足，但有效稿件都有同一结构：具体物理麻烦、补救方案失败、重复劳动产生代价、结尾保留可见未解决状态。

## 五、题材结论

### 校园尴尬与误会

这是唯一出现稳定信号的题材：

- A 字符加权人工率：42.94%
- B：42.71%
- C：48.10%
- B 单篇中位数最高：44.13%
- C 有 3/8 篇达到 60%，并在 CAMPUS-03 重复胜出。

### 恋爱日常与关系试探

当前方案整体失败，尤其 C：

- A 加权人工率：10.19%
- B：7.53%
- C：1.20%

低冲突恋爱场景被模型写成均匀的氛围、体感和停顿。继续增加因果字段只会让它更像结构化产品。应采用 **B-short**：350–550 字，隐藏消息全文，只保留一个物件或请求，第一次关系承诺成立就停止。

### 危机或具体任务

没有全局赢家：

- A 加权人工率：18.14%
- B：13.94%
- C：8.45%

漏水处置有成功信号，但包裹悬疑、密室、找人等场景大量落入通用悬疑和程序化能力展示。它们需要独立 D 路线，而不是再给 C 填四个字段。

## 六、收敛方案

### 路由 C：物件驱动的误认与追索

触发条件：

- 一个具体物件能证明或拆穿判断；
- 人物存在错误判断或隐蔽目的；
- 行动会改变物件位置、归属、完整性或可见状态；
- 对方或现实能形成反制。

WriterBrief 保留 C 的四个增强字段，但每项只允许一句，禁止抽象关系总结。

### 路由 A-lite：多人信息不对称与残缺消息

新增一个精简投影，不恢复完整 Planner：

- `secondary_actor_goal`
- `secondary_actor_observed_evidence`
- `competing_interpretation`
- `likely_countermove`
- `reader_uncertainty_to_preserve`

用于便签、误读、流言、多人在场、对方不能直接解释的场景。

### 路由 B-short：低冲突恋爱

- 目标 350–550 字；
- 不展开聊天记录、信件或回忆全文；
- 最多一个感官细节链；
- 只完成一个请求、接受、拒绝或承诺；
- 关系状态一旦通过动作成立，立即结束；
- 禁止结尾后再补“他没走/她没退/空气安静”等第二次确认。

### 路由 D：模板化悬疑与任务

必须包含：

- 一个可犯错的局部判断；
- 一个因面子、伦理或风险作出的不理想选择；
- 一个立刻可见的后果；
- 一个具体新限制。

减少钟表、脚步、灯光、雨声、阴影等通用悬疑信号，避免主人公像流程图一样正确处置所有问题。

## 七、下一轮实验

不要再次做 A/B/C 全局平均。使用本轮 11 篇正样本及同场景最低分稿作负样本，运行路由实验：

- A-lite：CAMPUS-04 类型 6 个新场景；
- C：CAMPUS-03 类型 6 个新场景；
- B-short：ROMANCE-02 类型 6 个新场景；
- D：悬疑/任务失败类型 6 个新场景；
- 每场景 3 次采样，共 72 篇。

验收从“所有题材同一个 Prompt 中位数 60%”改为：

1. 每条路由在对应场景的人工率中位数 ≥60%；
2. 至少 2/3 场景优于当前最佳基线；
3. 人工盲评不下降；
4. 同一场景三次采样中至少两次达标，避免靠一次满分抽奖冒充稳定性。

## 最终判定

这轮实验没有证明 B 或 C 可以全局达到 60%，但已经找到了两个可重复的真实信号：

- **物件误认/追索场景 → C**
- **残缺信息/多人意图误读场景 → A-lite**

因此项目应从“寻找唯一万能 WriterBrief”转向“按叙事机制选择低污染投影”。这不是退缩，反而是第一次从 72 篇结果里得到可执行的生成路由。
