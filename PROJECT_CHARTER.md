# Novel Workbench 项目主线

## 最终目标

建立一个本地单用户小说工作台，通过可观测、可替换、可比较的多阶段生成管线，提高 AI 小说正文质量。

## 核心叙事目标

人物只能依据已经知道、看到或听到的信息行动。

每个主要场景应尽可能形成：

```text
有限证据
→ 人物判断
→ 存在其他可选行动
→ 人物选择
→ 具体代价或承诺
→ 选择专属后果
→ 新的行动限制
→ 停在可见事实
```

不得用旁白替读者解释人物性格、心理结论或关系变化。

## 固定主管线

```text
Planner
→ Writer
→ Critic
→ Reviser
→ Judge
```

## 阶段职责

**Planner：**
确定场景目标、信息边界、人物选择、因果后果、新约束和停止事实。

**Writer：**
忠实执行 Planner，将规划转成可读正文，不重新规划。

**Critic：**
对照 Planner 和正文，定位缺失、缩水、冲突和越界，不负责改写。

**Reviser：**
只根据有效 Critic issues 做最小修订，保护正文已有优点。

**Judge：**
比较原稿与修订稿，判断修订是否真正改善，而不是仅仅变得平滑。

## 永久限制

- 不新增与主线并行的工作流。
- 不让一个阶段吞并其他阶段职责。
- 不因单次实验失败重建架构。
- TGbreak 仅为 Writer 可选模式，当前冻结。
- Critic Evidence / Compiler 实验功能冻结，不进入默认管线。
- 所有真实模型调用必须有明确实验问题和停止条件。
- 候选不得自动覆盖或自动选择。
