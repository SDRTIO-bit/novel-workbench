# DETECTOR_GENERALIZATION_V2: 叙事行为版 C 路径

## 目的

验证一个可用于小说生产的最小 Writer 改动：让 Brief 中的因果信息在正文里成为可见的行动、物件、代价和关系反应，而不是进一步扩张 Planner 或针对任何检测器调参。

## 组别

- **A：完整 Planner 基线**：沿用现有 Writer 指令；完整 Planner JSON 作为 Writer 输入。
- **B：WriterBrief 基线**：沿用现有 Writer 指令；当前 WriterBrief 作为 Writer 输入。
- **C：叙事行为增强 Brief**：使用 WriterBrief 与四个既有扩展字段：`available_causal_objects`、`rejected_alternative`、`cost_or_commitment`、`counteraction_or_disproof`。Writer 额外接收一段短的叙事行为指令。

除 C 的输入编译和该段短指令外，三组使用相同模型、场景、目标字数、上下文、Planner 结果和采样参数。

## C 组 Writer 指令的写作意图

该指令只表达小说技法，不提及 AI、检测、分数或规避。它要求：

1. 将关键选择落实为人物对现场可见物件、位置或身体动作的处理；
2. 让一个本可采用的替代方案在行动中被放弃，而非用解释性独白交代；
3. 让承诺或代价体现为时间、空间、资源或关系状态的具体变化；
4. 让关键行动引出其他人物可观察的反应或反制；
5. 在一个新发生、可验证的事实处结束场景，不用总结性判断代替事件。

这些要求不得凌驾于既有角色、事实边界、因果约束或安全规则之上。

## 实验规模与交付

- 12 个固定场景：校园尴尬与误会 4 个、恋爱日常与关系试探 4 个、危机或具体任务 4 个；
- 每组每场景生成 2 次，共 72 篇；
- 模型为 `deepseek-v4-pro`，关闭思考模式；正文上限 4,000 tokens；
- 运行产物保留可追溯映射，同时导出一个没有组别标记的单一 TXT，供人工复制至朱雀进行分段检测。

## 判读边界

检测报告的任意分段不保证等于一篇样本，因而仅作为段落层面的定性线索。正式的 A/B/C 比较须基于可恢复样本边界的数据；人工盲评与事实越界检查仍是小说质量的必要约束。
